Telegram Trading Bot

Description

The goal of this project is to interact with the Binance Blockchain to buy and sell newly created cryptocurrencies.

The method is to collect data about newly created currencies, the data is automatically posted on Telegram and can be collected with their API, Telethon.
That data is then processed, and automatically posted on honeypot.is to check if itâ€™s a scam token, the result is then webscrapped, and the bot will buy the token, follow its price until it sells it again, when some price target is reached.
The whole process is Illustrated here in a activity diagram.

Process

I broke the task up in modules, then researched for the API's and frameworks with the best documentation. Then i went on to with piecing them all together in the main loop, one module at a time.
Lots of hidden problems existed, in working with asynchronous code, strange transaction logs and so on.
But the debugging tool became my good freind by the end of the project.

Features:

  • The bot knows exactly how many crypto tokens you bhought, by looking into the metadata of the transaction, and extracting the information from the nested json objects and then converting it from hexadecimals to decimals.

  • Utilizes browerless chrome and selenium web automation to post info and webscrape results. Works locally and in the cloud, on Heroku, by using buildpacks with links to githup repositories.

  • Using Moralis API to recieve price updates.

  • Chrash resistant by storing the purchase information in a SQLlite database.

  • Trailing stoploss feature so the bot will follow the price of the tokens and sell them when the price has dropped a fixed percentage from the highest price recorded since they where purchased.

Conclusion:

The project is a minimal viable product and was just an attempt to make a codebase from where I can build ontop.

I do not dare to run this bot and its just created for the sake learning and having fun trying to solve complex problems.
