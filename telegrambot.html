Telegram Trading Bot

Description

The goal of this project is to interact with the Binance Blockchain using the Web3 python framework to be able buy and sell newly created cryptocurrencies in the cloud.

The method is to collect data about newly created currencies, which is posted automatically on Telegram with their API.
That data is then processed, and automatically posted on honeypot.is to check if itâ€™s a scam token, the result is then webscrapped, and the bot will buy the token, follow its price until it sells it again, when some price target is reached.
The whole process is Illustrated here in a activity diagram.

Features:

  • The bot knows exactly how many crypto tokens you buy per $ by looking into the metadata of the transaction, and extracting the information from the nested json objects and then converting it from hexadecimals to decimals.
  • Utilizes browerless chrome with selenium web automation to post info and webscrape results. Works locally and in the cloud by add buildpacks with links to githup reposotories.
  • To be able to run the bot on Heroku while using browserless chrome I found out that adding buildpacks with links to github repositories the drivers would be installed in the cloud.
  • Using Moralis API to recieve price updates on tokens bhought
  • Chrash resistant by storing the purchase information in a SQLlite database.
  • Trailing stoploss feature so the bot will follow the price of the tokens it bhought and sell when the price has dropped a fixed percentage from the highest price recorded.

Improvements:

The project is a minimal viable product and was just an attempt to make a starting point from where I can build on the code to make different trading bots and other application interacting with a blockchain. I do not dare to run this bot and its more for learning the framework.

  • The use of Asynchronous functions is not done correctly, as i dont await the data i just while loop and check whether is there.
